# Startup Radar Template

A personal daily scanner that finds startups matching your criteria (industry, funding stage, location) and flags open roles that fit your target job titles. Pulls from free public sources out of the box — no API keys required. Optional Gmail integration for curated VC newsletters.

Designed to be forked, cloned, and customized in minutes using Claude Code.

> Built by [Xaviera Ho](https://github.com/xavierahojjx-afk) — originally as a personal Stanford MBA job search tool, then generalized for public use.

---

## What you get

- **RSS + Hacker News + SEC EDGAR Form D** — free sources, no auth
- **Config-driven filtering** — roles, locations, industries, funding stage, seniority exclusions, all in one `config.yaml`
- **SQLite storage** + **Streamlit dashboard** to browse, mark status, and find warm intros
- **LinkedIn connections import** — flag startups where a 1st-degree connection works
- **DeepDive skill** — generate a branded one-page company research brief (.docx) scored against your fit criteria
- **Scheduling** — GitHub Actions, cron, Task Scheduler, launchd templates included
- **Optional Gmail** — pull curated VC newsletters (StrictlyVC, Term Sheet, etc.) with per-newsletter parsers generated by Claude Code

## Quickstart

```bash
git clone https://github.com/xavierahojjx-afk/startup-radar-template.git
cd startup-radar-template
pip install -r requirements.txt
```

Then launch Claude Code and run the setup skill:

```bash
claude
```

Inside Claude Code, type:

```
/setup-radar
```

The setup skill will interview you about your preferences, write your `config.yaml`, walk through optional integrations, and help you pick a scheduling method. Expect 5–15 minutes depending on how many optional features you enable.

Once configured, run the pipeline manually to test:

```bash
python main.py
```

Open the dashboard:

```bash
streamlit run app.py
```

## Sources

| Source | Auth? | Description |
|---|---|---|
| RSS (TechCrunch, VentureBeat, Crunchbase News) | None | Public feeds, always on by default |
| Hacker News (Algolia API) | None | Searches for "raised Series X" threads |

## Skills

## Scheduling

Pick one:

- **GitHub Actions** (recommended) — `.github/workflows/daily.yml`. Runs in the cloud, free, zero local setup. Works even when your PC is off.
- **Windows Task Scheduler** — see `scheduling/windows_task.md`
- **macOS launchd** — see `scheduling/launchd.plist.template`
- **Linux cron** — see `scheduling/crontab.example`
- **Manual** — just run `python daily_run.py`

## Customizing

Everything is meant to be edited:

- `config.yaml` — your preferences
- `app.py` — the Streamlit dashboard. Rearrange tabs, add views, drop features.
- `sources/` — add a new source by dropping a file here and wiring it into `main.py`
- `.claude/skills/deepdive.md` — tweak the research brief template
- `filters.py` — custom filter logic beyond what YAML supports

The intended workflow: fork, clone, run `/setup-radar`, then use `claude` to extend whatever you want. The point of this template is that Claude Code writes the "last mile" — custom parsers, custom filters, custom dashboard views — so you don't have to.

## Optional: Gmail source

If you subscribe to VC newsletters (StrictlyVC, Term Sheet, Venture Daily Digest, a16z Build, etc.) they're a gold mine of curated signal. To enable:

1. Create a Google Cloud project at https://console.cloud.google.com/
2. Enable the Gmail API
3. Create OAuth Desktop app credentials
4. Download as `credentials.json` into the repo root (never commit it — already in `.gitignore`)
5. In Gmail, create a label (e.g. "Startup Funding") and set filters so newsletter senders auto-label
6. Enable `sources.gmail` in `config.yaml`
7. First run will open a browser for consent and cache `token.json`

Inside Claude Code, `/setup-radar` walks you through this.

## Logs & troubleshooting

When the pipeline runs via `python daily_run.py` (or any of the scheduling templates), it writes one log file per day to `logs/YYYY-MM-DD.log`. The log captures every step: which sources fired, how many candidates each returned, what the filter rejected, what got written to the database, and any errors with full tracebacks.

If something looks wrong, start here:

```bash
# Latest log
cat logs/$(date +%Y-%m-%d).log

# All logs, most recent last
ls -lt logs/
```

Common patterns to grep for:

| You see | Likely cause |
|---|---|
| `Daily run failed: ... token ...` | Gmail OAuth expired — delete `token.json` and run `python main.py` interactively to re-auth |
| `RSS error (FeedName): ...` | Feed URL changed or temporarily down — check the feed in a browser |
| `EDGAR error: ...` | SEC rate-limited the User-Agent — usually transient, next run recovers |
| `0 candidate(s)` on every source | Your filters may be too strict — loosen `targets.industries` or `min_stage` in `config.yaml` |
| `No new startups to add` but you expected some | Candidates matched but got filtered — check the extraction lines above for `After filter:` |

Running `python main.py` directly (instead of `daily_run.py`) prints everything to the console in real time — useful when you're actively debugging.
