# /research — company research brief

Replaces: `/deepdive`.

## 1. Preflight

- Read `config.yaml`. Schema of record: `startup_radar/config/schema.py` (pydantic `AppConfig`) — trust the schema if answers drift.
- Extract:
  - `user.name`, `user.background`
  - `targets.roles`, `targets.locations`, `targets.industries`, `targets.min_stage`
  - `deepdive.fit_factors` (weights per factor: high / medium / low)
  - `deepdive.tier1_vcs` (list)
  - `deepdive.thresholds` (default strong=7.5, moderate=5.0)
- Missing `config.yaml` → route to `/radar` (Onboard). STOP.

## 2. Web research

Search: Crunchbase · Tracxn · Sacra · company's own site · recent press.

| Field | Source priority |
|---|---|
| Product + target market | company site → Crunchbase |
| Founders + backgrounds | LinkedIn → Crunchbase → press |
| Funding history (rounds, amounts, dates, leads) | Crunchbase → SEC EDGAR → press |
| Investors (flag "tier 1" if in `deepdive.tier1_vcs`) | round announcements |
| Competitors | Sacra → Tracxn → news |
| Role-fit signals | careers page → LinkedIn "we're hiring" |

## 3. Score + label

Apply `deepdive.fit_factors` weights to: industry_match · funding_stage · location · role_fit_signals · founder_pedigree · vc_tier. Sum → /10. Apply `deepdive.thresholds` → **STRONG** (≥strong) · **MODERATE** (≥moderate) · **WEAK**.

## 4. Rationale

Fit paragraph MUST reference:

- `user.background` — specific phrasing, not generic.
- `targets.roles` — exact matches the company is hiring for.

> RIGHT: "Fits your 6-year backend infra background; they're hiring a founding backend engineer in NYC."
> WRONG: "Looks like a strong fit for your profile."

## 5. Save investors (mandatory)

```
reports/<CompanyName>_investors.json
```

JSON list of investor company names. The dashboard's tier-2 warm-intro lookup greps this path.

## 6. Emit .docx (`python-docx` — already a project dep)

Path: `reports/<CompanyName>_Research_Brief.docx`.

Sections (in order):

1. **Header** — name, tagline, navy background, website + date subheader.
2. **Company Overview** — info table (HQ | Founded | Stage | Total Raised | Employees | Website) + 2–3 sentence description + target market.
3. **Founders & Team** — per founder: name, role, background.
4. **Funding & Investors** — table (Round | Amount | Date | Lead) + investors list; tier-1 highlighted.
5. **Competitors** — brief list.
6. **Role Fit Signals** — bullets referencing `targets.roles`.
7. **Fit Score** — two-column box: left = score/10 + label (color-coded), right = rationale.
8. **Watch Out For** — 2–3 honest risks or gaps.
9. **Footer** — "Generated by Claude · {date}".

**Design**

| Element | Value |
|---|---|
| Page | US Letter |
| Margins | 0.5" top, 0.75" sides + bottom |
| Font | Arial |
| Navy (headers, table bands) | `#1B3A6B` |
| Accent blue | `#2E75B6` |
| Strong label | `#1A7A4A` green |
| Moderate label | `#B45309` amber |
| Weak label | `#991B1B` red |

## 7. Report

Tell user the .docx path. Offer to open. Mention: dashboard "Warm intros" tab now works for this company (via the investors JSON).

## Constraints

- NEVER fabricate funding numbers, founder bios, valuations. "Not disclosed" if absent. Credibility > completeness.
- NEVER score without referencing `user.background` — generic scoring is the failure mode.
- NEVER skip the investor JSON save — dashboard lookup depends on it.
- NEVER change the output path format — warm-intro greps on `reports/<Co>_investors.json` and `reports/<Co>_Research_Brief.docx`.
- `startup-radar deepdive <Company>` CLI command exists but this skill is the canonical path — Claude's web research is richer than the CLI's stub.
