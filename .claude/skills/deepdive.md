# DeepDive Skill

## Trigger
User types `/deepdive [Company Name]` ‚Äî e.g. `/deepdive Anthropic`.

## What this does
Researches the named company on the public web and generates a branded one-page research brief as a `.docx` file. The fit score and rationale are tailored to the user's criteria from `config.yaml`.

## Before starting
1. Read `config.yaml`. Extract:
   - `user.name` and `user.background`
   - `targets.*` (roles, locations, industries, min_stage)
   - `deepdive.fit_factors` (weights)
   - `deepdive.tier1_vcs`
   - `deepdive.thresholds`
2. If `config.yaml` doesn't exist, tell the user to run `/setup` first and stop.

## Research steps
For the company the user named:

1. **Web search** across Crunchbase, Tracxn, Sacra, the company's own site, and recent news.
2. Gather:
   - What the company does (product, target market)
   - Founders and their backgrounds
   - Full funding history (rounds, amounts, dates, lead investors)
   - Key investors ‚Äî classify each as "tier 1" if they appear in `deepdive.tier1_vcs`
   - Competitors
   - Role fit signals ‚Äî are they hiring for any of the roles listed in `targets.roles`? Any public GTM/team expansion signals?
3. **Score** the company out of 10 using the weights in `deepdive.fit_factors`. High-weight factors contribute more. Apply thresholds from `deepdive.thresholds` to assign a label: STRONG / MODERATE / WEAK.
4. **Write a fit rationale** that references the user's specific background (from `user.background`) and their target roles ‚Äî not generic filler.

## Save investor names for warm-intro lookup
After research, write the list of investor company names to `reports/<CompanyName>_investors.json` so the Streamlit dashboard's tier-2 intro lookup can use them.

## Generate the .docx
Use `python-docx` (already in `requirements.txt`).

### Sections (in order)
1. **Header** ‚Äî Company name, tagline, navy background, website + date subheader
2. **Company Overview** ‚Äî Info table (HQ, Founded, Stage, Total Raised, Employees, Website) + 2‚Äì3 sentence description + target market
3. **Founders & Team** ‚Äî Each founder: name, role, background
4. **Funding & Investors** ‚Äî Table of rounds (Round | Amount | Date | Lead) + key investors list with tier 1 highlighted
5. **Competitors** ‚Äî Brief list
6. **Role Fit Signals** ‚Äî Bullets referencing the user's target roles from `config.yaml`
7. **Fit Score** ‚Äî Two-column box: left = score /10 + label (color-coded), right = rationale paragraph tailored to user
8. **Watch Out For** ‚Äî 2‚Äì3 honest risks or gaps
9. **Footer** ‚Äî "Generated by Claude ¬∑ {date}"

### Design
- US Letter, margins 0.5" top / 0.75" sides and bottom
- Font: Arial
- Navy: `#1B3A6B`, accent blue: `#2E75B6`
- Fit score color: green `#1A7A4A` (strong), amber `#B45309` (moderate), red `#991B1B` (weak)
- Output file: `reports/<CompanyName>_Research_Brief.docx`

## After generation
- Tell the user the file path
- Offer to open it
- Remind them they can now use the Streamlit dashboard's "Warm intros" tab for this company to see tier-2 connections via investors

## Keep it factual
If you can't find something (e.g. valuation), write "Not disclosed" ‚Äî don't fabricate. The brief's credibility matters.
